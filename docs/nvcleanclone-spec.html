NVClone — A clean-install tool for NVIDIA drivers

NVClone is a full-fidelity, drop-in functional clone of TechPowerUp NVCleanInstall. It takes a downloaded NVIDIA driver .exe, strips unwanted components and telemetry, optionally re-signs the modified package, and installs only the display driver (plus user-selected extras) onto Windows 11 — without GeForce Experience, without the NVIDIA App, and without background telemetry services.

This document is the complete, build-ready specification. Every behavior is described concretely: exact folders, exact INFs, exact pnputil commands, exact registry keys, and exact acceptance criteria. There is no aspirational scope; anything not proven-feasible is listed in §3 Out of Scope.

Non-negotiable requirements

  1. Functional parity with NVCleanInstall's core loop: Load → Detect → Select → Trim → (optional Sign) → Install.
  2. Signature safety by default. The default install path must never require re-signing and must never break the WHQL catalog.
  3. Runs elevated, ships as one portable .exe.
  4. No telemetry of its own, no network calls except an optional, user-initiated driver download.

2. Design Principles

  • Deterministic. Same input package + same options ⇒ identical output package (byte-for-byte except timestamps).
  • Fail loud. Any unexpected state (unknown INF layout, missing file the catalog expects, unsupported driver branch) halts with a precise error before any system change.
  • Reversible-by-default where possible. Pre-install cleanup offers a restore point; post-install telemetry removal only deletes files that exist and records each action to the log.
  • No magic constants. Every registry key, service name, and file pattern is named in this spec, not discovered at runtime.

3. Scope

In scope FULL

  • Loading a local NVIDIA driver .exe (7z SFX) and extracting it.
  • Detecting driver version, DCH vs Standard, branch, supported GPUs (from INF).
  • Selecting/excluding components: Display Driver (core), PhysX, HD Audio, USB-C, Optimus, NVContainer, GeForce Experience / NVIDIA App.
  • Removing telemetry files, services, and registry entries (post-install, signature-safe).
  • Installing via pnputil (primary); SetupAPI P/Invoke documented as advanced path.
  • Optional self-signed certificate generation + catalog re-signing for INF-edited packages.
  • Optional driver download from NVIDIA's download endpoint.
  • Optional pre-install cleanup of existing NVIDIA driver packages + services.
  • Post-install power/quality tweaks (Prefer Maximum Performance, disable audio power-saving, etc.).
  • GUI wizard + silent CLI mode.

Out of scope N/A

  • Linux / macOS / Windows-on-ARM host — Windows 11 x64 host only.
  • AMD / Intel GPU drivers. NVIDIA-only.
  • Direct DDU-style display-driver-uninstaller functionality (we can remove prior NVIDIA packages, but we are not a general uninstaller for third-party software).
  • NVIDIA Studio vs Game Ready branch auto-selection. User picks which .exe to load.
  • Mobile GPU (OEM INF) modding / device-ID injection. We install the package as NVIDIA ships it for the GPUs it declares; we do not rewrite INF device lists.
  • Cloud sync, accounts, profiles. None.

5. Technology Stack

The dominant constraint is that this is a privileged system utility doing deep Win32 interop, shipped as a single self-contained executable. That constraint selects the stack.

5.2 Why WPF and not WinUI 3

Decision record

WinUI 3 is the correct choice for consumer-facing, modern-design Windows apps. It is the wrong choice here for four concrete reasons:

  1. Elevation. MSIX-packaged WinUI 3 apps cannot run elevated. Unpackaged WinUI 3 can, but the bootstrap-API init path inside an elevated context is fragile. A driver installer must be elevated and must be unpackaged/portable.
  2. Win32 interop density. This app is ~60% Win32 calls (SetupAPI, ADVAPI32, registry, service control). WPF's interop story is a decade more mature; WinUI 3's HWND-interop adds friction for nothing we gain.
  3. Single-file publish. WPF self-contained single-file is rock-solid and small. WinUI 3 single-file is possible but heavier and historically finicky with the WinAppSDK runtime.
  4. UI complexity is low. The app is a wizard with checkboxes, a few combo boxes, and a progress log. There is no win from Fluent/mica/acrylic that justifies the interop cost.

WPF on .NET 10 is in maintenance mode, which is exactly the correct posture for a system utility: stable, supported for 3 years, no churn.

8.1 INF files of interest

  • Display.Driver\nv_dispi.inf — primary DCH consumer INF.
  • Display.Driver\nvami.inf — reference desktop.
  • Display.Driver\nvdmi.inf, nvmi.inf, nvcvi.inf — OEM variants.
  • HDAudio\nvhda.inf — HDMI audio.

NVClone picks the active INF by matching the installed GPU's PCI vendor/device ID against each INF's [Models] section (see §13).

9. Component Exclusion Logic

Each component is defined as a data row (not hardcoded logic). The map below is the authoritative definition.

ComponentDefaultFolders included when selectedInstall mechanism
Display Driver REQUIREDOn (locked) Display.Driver\ pnputil /add-driver <activeINF> /install
PhysXOff PhysX\ msiexec /i PhysX_*.msi /qn + verify HKLM\SOFTWARE\AGEIA Technologies
HD AudioOff HDAudio\ pnputil /add-driver HDAudio\nvhda.inf /install
OptimusOff (auto-on if laptop detected) Display.Optimus\ pnputil /add-driver Display.Optimus\<inf> /install
NVContainerOff Display.NvContainer\ (minus telemetry files — §10) pnputil /add-driver Display.NvContainer\<inf> /install; service config tuned post-install
USB-C / MSTOff USB-C subtree within NvContainer or top-level USBC\ if present pnputil /add-driver <inf> /install
GeForce ExperienceOff, always excluded none Folder deleted from working tree
NVIDIA AppOff (opt-in) App\ / NVIDIA App\ Launched as separate .exe installer (no INF)

Exclusion algorithm (Build phase, §15): for every component NOT selected, delete its folder(s) from the extracted working tree, then remove the corresponding Include/Needs references from the primary INF only if the "Edit INFs (advanced, requires resign)" option is on. Default = folder deletion only, which keeps the NVIDIA catalog valid.

Signature invariant

Deleting an entire component folder does not invalidate the display-driver .cat catalog as long as the display-driver INF's [SourceDisksFiles] does not list files from that folder. NVIDIA's structure satisfies this for PhysX/HDAudio/Optimus/NvContainer because each is its own self-contained package with its own INF/catalog. Telemetry DLLs located inside Display.Driver\ are the exception — those are referenced by the display INF and must be handled post-install (§10), not by source-tree deletion.

10. Telemetry Removal Targets

Telemetry removal runs post-install against the live system + DriverStore, never against the source INF (to preserve signature). Targets are enumerated; NVClone deletes only what exists and logs each.

10.2 Services (stop + disable + delete — only the telemetry/helper ones, never the core)

If the user separately enables "Disable NVContainer" (advanced), the consequence (NVCP requires the Store version) is shown in a confirmation dialog first.

10.3 Registry

HKLM\SOFTWARE\NVIDIA Corporation\Global\FTS\  → set "EnableRIDefinition" / "RID" / "RISENabled" = 0 (if present)

All writes use RegistryView.Registry64 and wrap each operation in try/except with structured logging; missing keys are not errors.

11. Driver Installation Mechanics

11.1 Primary mechanism — pnputil

For each selected component with an INF:

pnputil.exe /add-driver "<WorkDir>\<ComponentFolder>\<inf>" /install
  • Launched as a child Process with Verb not set (already elevated), CreateNoWindow=true, redirected stdout/stderr, UTF-8.
  • Exit codes: 0 = OK; 3010 = ERROR_SUCCESS_REBOOT_REQUIRED (surface reboot prompt); anything else = hard fail with full captured output in the log.
  • Published-name parsing: parse stdout for lines matching Published Name:\\<...>\oemNN.inf (regex Published Name:\s*(\\.*?oem\d+\.inf)). Stored per-component for later cleanup (§18).
  • Timeout: 600s per INF; on timeout, kill + fail loud (do not leave a half-staged package).

11.2 Advanced mechanism — SetupAPI P/Invoke

Documented and stubbed; enabled behind --use-setupapi. Calls DiInstallDriverW(hwndParent, InfPath, flags, &needReboot) via newdev.dll. Returns the same semantics. Use when pnputil output format drifts (defensive), or for binding a specific INF to a specific device instance via DiInstallDevice.

11.3 PhysX (non-INF component)

msiexec.exe /i "<WorkDir>\PhysX\<PhysX_*_SystemSoftware.msi>" /qn /norestart INSTALLDIR="<ProgramFiles>\NVIDIA Corporation\PhysX"

Exit code 0 = success. MSI log captured to %TEMP%\nvclone-physx.log.

11.4 NVIDIA App / GFE (opt-in only)

If the user explicitly opts in, run the component's own installer .exe with its documented silent switch (typically -s). Default = excluded entirely.

11.5 Reboot policy

If any component returned 3010 OR if a prior cleanup required reboot, prompt: "Reboot now / Reboot later / Skip." Never auto-reboot without consent (except in CLI mode with --reboot).

12. Feature: Package Load & Extract

Inputs

  • Path to NVIDIA driver .exe (file picker, drag-drop, or CLI arg).

Steps

  1. Validate file exists, > 50 MB, has PE header AND a trailing/embedded 7z signature. Read first 2 bytes (MZ) and scan for 7z magic 37 7A BC AF 27 1C.
  2. Extract 7za.exe from embedded resource to %TEMP%\nvclone\7za.exe (LGPL notice shown in About).
  3. Working dir: %TEMP%\nvclone\work\<sha256-of-exe-path[:12]>\ — cleared on start, reused across pages, deleted on app exit.
  4. Run: 7za.exe x "<driver.exe>" -o"<WorkDir>" -y -aoa
  5. Verify extraction: WorkDir\Display.Driver\ exists and contains ≥1 .inf + ≥1 .cat. Else → fail with "Not a recognized NVIDIA driver package."

Acceptance

  • Loading the same .exe twice yields identical working trees.
  • A non-driver 7z SFX is rejected with a precise message before any UI state changes.
  • Disk full during extraction → partial cleanup + clear error.

13. Feature: Package Detection & Parsing

Outputs

  • Driver version (e.g. 600.84), branch, WHQL/DCH flag, build date.
  • List of present components (folders actually in the package).
  • Active INF + matching GPU (if any NVIDIA adapter present).
  • Supported PCI device IDs (from ListDevices.txt + INF [Models]).

Steps

  1. Version: parse NVI2\setup.cfg [CheckMe] / [Version] for DriverVer; cross-check against Display.Driver\nv_dispi.inf [Version] DriverVer.
  2. DCH vs Standard: presence of Display.NvContainer\ ⇒ DCH. Absent + presence of nv_disp.inf ⇒ Standard. Surface in UI.
  3. Active INF: enumerate installed NVIDIA adapters via SetupDiEnumDeviceInfo + SetupDiGetDeviceRegistryProperty(SPDRP_HARDWAREID). Match the PCI ID (e.g. PCI\VEN_10DE&DEV_2484&SUBSYS_...) against each candidate INF's [Models]/[NvidiaDevices] section. First match = active INF.
  4. Components present: scan top-level folders; intersect with the component map (§9).

Acceptance

  • If no NVIDIA adapter is present, install still proceeds (use first INF) but a warning is shown.
  • DCH/Standard detection is correct for ≥95% of shipped drivers since branch 430 (validated against a fixture set).

14. Feature: Component Selection UI

Wizard page 3. One checkbox per component from §9. Defaults:

  • Display Driver: on, locked (cannot uncheck — core requirement).
  • PhysX, HD Audio, NVContainer, USB-C, NVIDIA App: off.
  • Optimus: auto-on if laptop (battery detected via GetSystemPowerStatus).
  • GeForce Experience: hidden / forced-off (we don't show a checkbox; we just never include it).

Additional options panel:

  • ☑ Remove telemetry (default on).
  • ☑ Disable NVIDIA HD Audio power-saving (default off).
  • ☑ Prefer Maximum Performance (default off).
  • ☐ Disable NVContainer (advanced; warns NVCP needs Store version).
  • ☐ Clean previous driver before install (default off).
  • ☐ Edit INFs (advanced; requires re-sign — see §16).
  • ☐ Export package only (don't install).
  • Language selector (default = OS UI language; falls back to English).

15. Feature: Package Build (Trim / Strip)

Inputs

Extracted working tree + selection state.

Steps

  1. Compute the excluded folder set from selection.
  2. Delete each excluded folder from WorkDir.
  3. Telemetry files inside Display.Driver\: do not delete here (would break catalog). Marked for post-install removal.
  4. If Edit INFs on: rewrite the active INF's [SourceDisksFiles], [NvidiaDevices.CopyFiles], [CoInstallers], and telemetry-named copy lines to drop deleted files; then regenerate .cat with inf2cat and re-sign (§16).
  5. If Export on: robocopy/Directory.Copy to chosen output path; stop.
  6. Else: hand off WorkDir to Install phase.

Acceptance

  • Build with all defaults produces a tree containing only Display.Driver\.
  • Build is deterministic given identical input + options.
  • Without Edit-INFs, no .cat/.inf is modified.

16. Feature: Driver Signing (Advanced)

Only required when

The user enabled Edit INFs in §14, which modifies the display INF and invalidates the WHQL catalog. The default flow (folder-deletion only) does not need this.

Steps

  1. Generate self-signed code-signing cert (ECC P-256, CN=NVClone Self-Sign, 1-year validity) via CertificateRequest.Create + X509Certificate2.
  2. Install into Cert:\LocalMachine\Root and Cert:\LocalMachine\TrustedPublisher (so Windows trusts the re-signed catalog during staging).
  3. Regenerate catalog: inf2cat.exe /driver:"<WorkDir>\Display.Driver" /os:10_x64.
  4. Re-sign catalog: signtool sign /fd SHA256 /sha1 <thumbprint> "<WorkDir>\Display.Driver\nv_dispi.cat" (signtool bundled as optional sidecar) OR PowerShell Set-AuthenticodeSignature -FilePath ... -Certificate $cert -HashAlgorithm SHA256.
  5. Record cert thumbprint + uninstall path in the log, so the user can remove the self-signed root later.

Acceptance

  • After sign, pnputil /add-driver succeeds without signature errors.
  • The self-signed root is removable from a one-click "Remove NVClone self-sign cert" action.

17. Feature: Download from NVIDIA

Optional page. NVIDIA has no stable JSON API; we scrape the documented download flow.

Steps

  1. User picks Product Type (GeForce/Quadro/Tesla), Series, Model, OS, Branch (Game Ready/Studio), language.
  2. GET https://www.nvidia.com/Download/processFind.aspx?psid=<psid>&pfid=<pfid>&osid=<osid>&lid=<lid>&whql=&lang=<lang> — returns HTML with the driver detail page link + download chk value.
  3. Parse the detail page for the downloadUrl JS var → resolves through processDriver.aspx → final *.exe on us.download.nvidia.com / international.download.nvidia.com.
  4. Stream to %TEMP%\nvclone\downloads\<suggestedName> with progress reporting via HttpClient.SendAsync + HttpCompletionMode.ResponseHeadersRead.
  5. Verify SHA-256 if NVIDIA publishes one on the detail page; warn (don't fail) on mismatch.
Maintenance note

NVIDIA can change their HTML at any time. The parser is isolated in Download/NvidiaDriverSearch.cs behind an interface, with two selector strategies (XPath + regex fallback). Failure to parse → user is told to download manually from nvidia.com/Download and load the local .exe (the primary path). This is the only "best-effort" surface in the spec and it degrades gracefully.

18. Feature: Previous-Driver Cleanup

Steps

  1. System Restore point: create via WMI CreateRestorePoint (best-effort; log if SR is disabled).
  2. Enumerate published NVIDIA driver packages: pnputil /enum-drivers, filter lines where Original Name starts with nv (covers nv_dispi.inf, nvhda.inf, nvami.inf etc.). Collect Published Name = oemNN.inf.
  3. Force-delete each: pnputil /delete-driver oemNN.inf /uninstall /force. Tolerate "in use" → request reboot.
  4. Services: stop NVDisplay.ContainerLocalSystem temporarily is NOT done (would black-screen). Instead only the helper services from §10.2 are touched.
  5. Files: delete %ProgramFiles%\NVIDIA Corporation\ subfolders that belong to removed components (NvBackend, Display.NvContainer plugins) — never Drs (NVCP profiles) or Control Panel Client.

Acceptance

  • Cleanup never leaves the display adapter without a bound driver for more than the reboot window.
  • Restore point exists or is logged as unavailable.

Same Core pipeline, no WPF. Invoked when any argument is present.

  list-gpus     Print detected NVIDIA adapters. No install.

21. Feature: Logging & Diagnostics

  • Location: %ProgramData%\NVClone\Logs\NVClone-YYYYMMDD-HHmmss.log (rolling; keep last 20).
  • Levels: Trace/Debug/Info/Warn/Error. Default Info; --verbose ⇒ Debug.
  • Captured: every pnputil/7za/msiexec command + full stdout/stderr + exit code; every registry/service/file action with before/after; INF edits as unified diff.
  • Privacy: scrub current username and machine name; never log the self-signed cert private key.
  • "Save diagnostics bundle" button zips the last log + pnputil /enum-drivers output + dxdiag (user-consented) for support.

22. Feature: Elevation & Bootstrapping

  • App manifest: <requestedExecutionLevel level="requireAdministrator" uiAccess="false"/>. UAC prompt on launch; no in-app re-elevation needed.
  • Single-instance via named mutex Global\NVClone_SingleInstance.
  • If launched with --wait-for-pid <n>, parent (unelevated) can spawn the elevated helper and read its exit code — used for "Run as admin" from a non-elevated shell.
  • Critical operations wrapped in a top-level try/catch that writes a crash dump path to %ProgramData%\NVClone\Crash\ and shows a dialog.

24. Test Strategy

24.1 Unit (xUnit, NVClone.Core.Tests)

  • PackageParser: feed fixture INFs (real, anonymized branches 4xx–6xx) and assert version/DCH/component extraction.
  • ComponentMap: assert every component folder maps 1:1 and unknown folders are ignored.
  • TelemetryTarget: assert regex matches for FileRepository paths.
  • pnputil output parser: golden outputs → published names + exit code mapping.
  • CLI arg parser: every option round-trips.

24.2 Integration (gated, manual or CI-with-VM)

  • Snapshotted Win11 VM; install a known driver via NVClone; assert Device Manager shows the new driver version, telemetry files absent, services absent, registry keys absent.
  • Round-trip: install → clean → install different version.
  • Reboot-required path (driver in use) returns exit 1.

24.3 Regression fixtures

One anonymized .exe-derived tree per supported major branch (current target: 580/590/600 series as of 2026-07). Stored in tests/fixtures/ (LFS, not in repo proper).

25. Risks & Mitigations

RiskLikelihoodMitigation
NVIDIA changes INF layout / adds a new folder.MediumUnknown folders are ignored, not fatal. Component map is data-driven; new entries added without recompiling logic. Unknown INF layout → halt with "unsupported package, please report."
NVIDIA download HTML changes.Medium-HighTwo parser strategies + graceful degrade to "download manually." Isolated behind interface.
pnputil output format changes in future Win11.LowExit-code is source of truth; stdout parsing only for published names. SetupAPI path is the fallback.
Self-signed cert lingers after uninstall.MediumOne-click cert-removal action; cert CN is fixed/identifiable; logged thumbprint.
User blackscreens mid-cleanup.Low (we don't stop core services)§18 explicitly avoids stopping NVDisplay.ContainerLocalSystem + nvlddmkm.
Telemetry DLL locked by running process.MediumStop the owning service first; if still locked, schedule delete-on-reboot via MoveFileEx(MOVEFILE_DELAY_UNTIL_REBOOT) and prompt reboot.
AV flags self-signed driver.Low-MediumDocument expected exclusions; cert is in TrustedPublisher. No EDR bypass attempted.
PhysX MSI fails on newer Windows.LowTreated as non-fatal; install continues; logged.
